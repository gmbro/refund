## 📖 서비스 소개

**환불원정대**는 부당한 위약금으로 고통받는 소비자를 위한 **AI 기반 소비자 분쟁 진단 서비스**입니다.

소비자가 겪고 있는 분쟁 상황을 입력하면, AI가 **소비자분쟁해결기준**과 **관련 법령·판례**를 자동으로 검토하여 **기초 법률 진단서**를 생성합니다. 이 진단서를 바탕으로 전문 변호사에게 바로 상담을 신청할 수 있는 원스톱 서비스입니다.

### 💡 왜 만들었나요?

- 많은 소비자가 업체의 과도한 위약금 요구에 제대로 대응하지 못합니다.
- 법률 상담의 문턱이 높아 포기하는 경우가 많습니다.
- **소비자분쟁해결기준**이라는 강력한 법적 근거가 있지만, 일반인이 이를 찾아 적용하기 어렵습니다.

**환불원정대**는 AI가 이 과정을 자동화하여, 누구나 자신의 법적 권리를 쉽게 확인하고 전문가 도움을 받을 수 있도록 합니다.

> 💡 15개 카테고리에 포함되지 않는 분쟁도 **"기타 (직접 입력)"**으로 진단이 가능합니다.

### 2. ⚖️ AI 기초 법률 진단

- **소비자분쟁해결기준**을 절대적 기준으로 적용한 자동 분석
- 업체 요구 위약금 vs 법적 기준 위약금 **비교 계산**
- 법적 기준 초과 위약금 **자동 감지 및 경고**
- 적용 법령 조항 **자동 인용**
- 계약서/이용약관 첨부 시 AI가 내용을 읽고 **불공정 조항 식별**

### 3. 🔢 카테고리별 환불액 자동 계산

각 카테고리의 법적 기준에 맞는 정확한 환불액을 산출합니다:

- **헬스장/PT**: `환불금액 = 총 계약대금 - 이용료 - 위약금(최대 10%)`
- **예식장**: 예식일까지 남은 일수에 따른 단계별 위약금 적용
- **숙박/항공**: 이용 예정일 기준 시기별 차등 위약금
- **의료/성형**: 횟수제 잔여분 비례 환급, 사업자 귀책 시 위약금 면제
- **기타 카테고리**: 소비자분쟁해결기준 일반 원칙 (위약금 10% 이내)

### 4. 📋 전문 변호사 상담 연결

- AI 진단 결과를 기반으로 **변호사 상담 즉시 신청**
- 고객 정보 + AI 진단서 + 변호사용 리포트 **자동 통합 전달**
- 영업일 1~2일 이내 전문 상담 안내

### 5. 🎙️ 음성 입력 지원

- Web Speech API 기반 **한국어 음성 입력**
- 길고 복잡한 상황 설명을 말로 편하게 입력 가능

### 6. 📄 계약서/이용약관 분석

- PDF, 이미지(JPG, PNG, WebP) 파일 첨부 지원 (3MB 이하)
- 이용약관 URL 입력 시 자동 크롤링 및 분석
- 소비자분쟁해결기준보다 불리한 조항 **자동 식별**

## 🔄 사용자 흐름 (User Flow)

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  STEP 1     │    │  STEP 2     │    │  STEP 3     │    │  STEP 4     │    │  STEP 5     │
│  카테고리    │───▶│  정보 입력   │───▶│  AI 분석    │───▶│  진단 결과   │───▶│  접수 완료   │
│  선택       │    │  (금액/일자) │    │  (로딩)     │    │ + 상담 신청  │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

**STEP 1** — 15개 카테고리 중 분쟁 유형 선택  
**STEP 2** — 결제 금액, 이용 기간/횟수, 위약금, 상황 설명 등 입력 + 계약서 첨부(선택)  
**STEP 3** — AI가 소비자분쟁해결기준 + 관련 법령을 분석 (로딩 화면)  
**STEP 4** — 진단 결과 확인 (법적 환불액, 초과 위약금, 쟁점 분석) + 변호사 상담 신청  
**STEP 5** — 접수 완료 안내

## 🚀 시작하기

### 사전 요구사항

- Node.js 18 이상
- npm 또는 yarn
- Supabase 프로젝트 (무료 플랜 가능)
- Google Gemini API 키

### 설치 및 실행

```bash
# 1. 의존성 설치
npm install

# 2. 환경변수 설정
cp .env.local.example .env.local
# .env.local 파일을 열어 아래 값을 설정하세요

# 3. 개발 서버 실행
npm run dev

# 4. 브라우저에서 확인
# http://localhost:3000
```

### 환경변수 설정

`.env.local` 파일에 다음 변수를 설정하세요:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your-supabase-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key

# Google Gemini AI
GEMINI_API_KEY=your-gemini-api-key

# korean-law-mcp (법령 검색 서버)
MCP_REMOTE_URL=https://korean-law-mcp.fly.dev/mcp

# 관리자 비밀번호 (Admin 페이지 로그인용)
ADMIN_PASSWORD=your-secure-admin-password
```

> ⚠️ **중요**: `GEMINI_API_KEY`와 `ADMIN_PASSWORD`는 `NEXT_PUBLIC_` 접두사를 사용하지 않습니다. 이 값들은 서버에서만 접근 가능하며 브라우저에 절대 노출되지 않습니다.

### 데이터베이스 설정

Supabase SQL 에디터에서 `supabase_setup.sql`을 실행하여 테이블과 RLS 정책을 생성합니다:

```bash
# 또는 Supabase CLI 사용
supabase db push
```

### 프로덕션 배포 (Vercel)

```bash
# Vercel CLI로 배포
npx vercel --prod
```

> Vercel 프로젝트 설정에서 환경변수를 반드시 등록해야 합니다.

## 🔐 관리자 시스템 (Admin)

### 접속 방법

`/admin` 경로에서 접속하며, 환경변수에 설정된 비밀번호로 인증합니다.

### 관리자 기능

| 기능 | 설명 |
|------|------|
| **접수 내역 조회** | 전체 상담 접수 내역을 시간순으로 조회 |
| **상태 관리** | `작성 중` → `상담 요청됨` → `상담 완료`로 진행 상태 변경 |
| **변호사 리포트** | AI가 생성한 사건 분석 리포트 상세 확인 |
| **고객 정보** | 연락처, 이메일, 입력 원문, 첨부 파일명 확인 |
| **로그아웃** | 세션 토큰을 즉시 폐기하고 로그아웃 |

### 관리자 인증 흐름

```
[Admin 로그인]
      │
      ▼
  POST /api/admin/login  ──── 서버에서 비밀번호 검증 (env var)
      │
      ▼
  세션 토큰 발급 (Base64)
      │
      ▼
  GET /api/admin/cases ──── Authorization: Bearer <token>
  PATCH /api/admin/cases ── 서버에서 토큰 검증 후 Supabase 조회/수정
```

> 💡 관리자 페이지는 Supabase를 직접 호출하지 않고, 서버 API를 통해 인증된 요청만 처리합니다.

## 🛡️ 보안 아키텍처

### 환경변수 분리

| 변수 | 접두사 | 노출 범위 | 설명 |
|------|--------|----------|------|
| `GEMINI_API_KEY` | 없음 | 🔒 서버 전용 | AI API 키, 브라우저에서 접근 불가 |
| `ADMIN_PASSWORD` | 없음 | 🔒 서버 전용 | 관리자 비밀번호, 소스코드에 미포함 |
| `MCP_REMOTE_URL` | 없음 | 🔒 서버 전용 | 법령 검색 서버 주소 |
| `NEXT_PUBLIC_SUPABASE_URL` | `NEXT_PUBLIC_` | 🌐 클라이언트 | Supabase 프로젝트 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `NEXT_PUBLIC_` | 🌐 클라이언트 | Supabase 익명 키 (RLS로 보호) |

> `NEXT_PUBLIC_` 접두사가 붙은 변수는 브라우저 번들에 포함됩니다. 이는 Supabase의 설계 원칙에 따른 것이며, RLS(Row Level Security) 정책이 실질적인 데이터 보호를 담당합니다.

### HTTP 보안 헤더

모든 페이지에 다음 보안 헤더가 자동 적용됩니다:

| 헤더 | 값 | 목적 |
|------|------|------|
| `X-Content-Type-Options` | `nosniff` | MIME 타입 스니핑 공격 방지 |
| `X-Frame-Options` | `DENY` | 클릭재킹(iframe 삽입) 공격 방지 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 외부 사이트로의 URL 정보 유출 방지 |
| `Strict-Transport-Security` | `max-age=63072000` | HTTPS 연결 강제 (2년) |
| `Permissions-Policy` | `camera=(), microphone=(self)` | 브라우저 권한 제한 (마이크만 자체 도메인 허용) |
| `Cache-Control` (API) | `no-store` | API 응답 캐시 비활성화 |

│  ✅ Supabase 조회/수정 (Admin API 경유)                        │
│  ✅ 변호사 리포트 생성 및 저장 (클라이언트에 미전달)               │
└────────────────────────┬──────────────────────────────────────┘
                         │
┌────────────────────────▼──────────────────────────────────────┐
│                    Supabase (PostgreSQL + RLS)                  │
│                                                               │
│  ✅ RLS 활성화로 anon key 접근 범위 제한                         │
│  ✅ 분석 결과, 고객 정보, 리포트 저장                             │
└───────────────────────────────────────────────────────────────┘

## 🛡️ AI 분석 파이프라인

분석 요청이 들어오면 아래 순서로 처리됩니다:

1. **환불액 계산** — 카테고리별 전용 계산 엔진으로 법적 환불액 산출
2. **법령 검색** — MCP 클라이언트로 관련 법령·판례·공정위 결정 검색 (실패 시 폴백 데이터 사용)
3. **이용약관 크롤링** — 사용자가 URL을 입력한 경우 약관 내용 자동 추출
4. **AI 진단서 생성** — Gemini가 모든 데이터를 종합하여 고객용 기초 진단서 작성
5. **변호사 리포트 생성** — 변호사가 사건을 빠르게 파악할 수 있는 전문 리포트 작성 (Admin 전용)
6. **DB 저장** — 모든 결과를 Supabase에 저장하고 고유 ID 반환

## ⚖️ 법적 고지

> **이 서비스는 법률 자문을 제공하지 않습니다.**

- 환불원정대는 소비자분쟁해결기준을 바탕으로 **참고용 정보**를 제공하는 AI 서비스입니다.
- AI가 제공하는 진단 결과는 입력 정보에 기반한 **단순 가이드라인**이며, 법적 효력이 없습니다.
- 구체적인 법적 판단과 대응은 반드시 **법률 전문가와 상담**하시기 바랍니다.
- 서비스 이용 시 수집되는 개인정보는 상담 연결 목적으로만 사용되며, 수집일로부터 1년 후 자동 파기됩니다.
